# Stage 8-10 敌人运动与发弹逻辑伪代码

本文总结重建工程中 Stage 8、9、10 新增的全部敌人、场景敌人、中 Boss、关底 Boss 及其内部节点。早期关卡复用敌人的具体行为由 Stage 1-7 文档说明；这里仍在覆盖表中列出，避免把“本关实际生成”和“本关新增实现”混为一谈。子弹生成后的通用运动、派生与销毁见 `projectile-pseudocode.md`。

## 记号

```text
D := clamp(difficulty, 0, 4)
DIFF[a,b,c,d,e] := 按 D 取表项
t := 当前实体 helperTimer；切换状态后通常归零
T := 父实体/根 Boss 的 helperTimer
cycle(n) := t mod n
ANGLE(a) := a & 0xffff
AIM(x,y) := 从 (x,y) 指向自机的 16 位角
TURN(a,target,step) := 沿最短圆周方向、每帧最多 step 地逼近 target
MOVE(a,v) := (x,y) += (cos(a),sin(a))*v
FAN(vis,id,pos,a,v,count,spread) := 以 a 为中心生成 count 发、总张角 spread 的扇形
RING(...) := FAN(..., spread=0x10000)
SHOT(...) := 生成单发
RAND(seed) := 原版脚本使用的确定性随机值
INSIDE(m) := 随自机横向偏移的 600x720 游戏区，向外扩 m
```

角度 `0x0000/0x4000/0x8000/0xc000` 分别表示右、下、左、上。`visual` 只决定弹图，`projectileId` 才决定子弹运动；本文按 `vis/id` 记录两者。除非另行说明，普通敌人 HP 归零后生成本类型的奖励和爆炸并停用，越过自己的外扩边界时静默停用。全局调度顺序是“敌人更新运动和状态 -> 敌人发弹 -> helperTimer 增加”，Stage 8 的专属敌人把发弹直接放在更新函数中。来源：`src/stage_runtime.cpp:4755-5066`、`src/stage_runtime_late.cpp:2150-2185,3152-3171`。

### `0x95`：间歇移动的量化狙击机

```text
每帧先 y += dispatchCount(1..4)
若进入可见区则可攻击
t==0: speed=1
若 t%30 > 19: MOVE(sourceAngle,speed)       # 每 30 帧只移动后 10 帧

cadence = DIFF[39,24,18,14,12]
shotSpeed = DIFF[5.5,6.5,7,7.5,9]
若 t%cadence==0，枪口 y<360、在屏内且距自机>50:
    a = AIM(枪口) 量化到 0x1000
    SHOT(9/0,a,shotSpeed)
    SHOT(9/0,a,shotSpeed+0.2)

HP<=0: 小型消散 + 1000 奖励；离开 INSIDE(100) 后退出
```

### `0x97/0x98/0x99`：旋转炮台族

三者共用瞄准炮台和三连窗口；区别仅是平移：`0x97` 原地，`0x98` 水平速度 1.5，`0x99` 初速 6 并在前 100 帧每帧减 0.06 至停止。

```text

    0x97 speed=0
    0x98/99 angle=朝舞台内侧，speed=1.5/6
    turretAim=AIM(x,y-31); mountAngle=0

    MOVE(angle,speed)
    mountAngle -= speed*522*左右符号
    liveAim=AIM(x,y-31)
    period=DIFF[90,60,44,60,44]
    step=DIFF[30,18,12,12,10]
    shotSpeed=DIFF[5.5,6.4,7,7,8]
    firingWindow := local=(t-30)%period < step*3
    非 firingWindow 时 turretAim 以 |delta|/10+0x30 追踪 liveAim
    firingWindow 且 local%step==0、旋转枪口在上半屏且距自机>50:

```

### `0x9a`：三枪口大型控制器

```text
state 0:
    y += dispatchCount
    到 y>=-140 时生成偏移 (0,+120)、半径120 的 0x151 共享碰撞体
    state=1，可攻击
```

### `0x9b/0x9c + 0x9d`、`0x9e/0x9f + 0xa0`、`0xa1/0xa2 + 0xa3`：四挂件复合机

每一对奇偶根类型行为相同；偶数尾型 `0x9c/0x9f/0xa2` 被击破时额外掉落奖励。三族的挂件、最低速度和弹图不同。

```text
族表:
    root 0x9b/9c -> child 0x9d, initialSpeed=11.4, minSpeed=2.4, rootVis=7
    root 0x9e/9f -> child 0xa0, initialSpeed=12.2, minSpeed=3.2, rootVis=9
    root 0xa1/a2 -> child 0xa3, initialSpeed=11.8, minSpeed=2.8, rootVis=7

state 0:
    第0帧把根放到左右场外 (-100或820,-100)，显示20帧标记
    t>=20: 生成四个对应 child；朝自机，state=1，可攻击

state 1:
    t<280:
        每50帧随机一个舞台目标
        前60帧 speed-=0.15，最低不低于族表 minSpeed
    t>=280: 目标改为 y=-500，speed+=0.1
    angle=TURN(angle,AIM(目标),0x280); MOVE(angle,speed)

    60<=t<280:
        period=DIFF[90,75,60,60,50]
        window=DIFF[25,40,40,40,40]
        local=(t-60)%period
        local==0 缓存 AIM；local<window 且 local%5==0:
            v=DIFF[7,8,9,8.5,9.5]+local*0.08
            SHOT(rootVis/0,cachedAim,v); D>2 再发 v+2

child:
    用 1/8 插值追随 root+固定偏移；0x9d 直接瞄准自机，
    0xa0/0xa3 用父 AIM 加构造角后平滑转向
    同一 60..280 窗口每4帧发弹，v=DIFF[7,8,9,8,9.5]+local*0.08
    D>2 再发 v+2.5

根 HP<=0: 根爆炸并立即退出；child 随父退出
t>240 且根离开 INSIDE(128): 静默退出
```

来源：`src/stage_runtime_enemy08.cpp:571-788`。

## 公共死亡/子部件语义

```text
0x137（PostDeathCounter）:
    普通敌人、可击部件或 Boss 击破均可创建，并非 Boss 专属。
    counterMode!=1: 立即退出。
    counterMode==1:
        固定在死亡位置；不可见、不移动、不可攻击。
        interval=DIFF[5,4,3,2,2]
        若 player.score!=0 且 t%interval==0 且 y<600:
            在死亡实体半径内随机取 point
            heading=AIM_FROM(point)+RAND[-1000,+1000]
            threshold=RAND[180,480]
            若 DIST(point,player)<=480 且 DIST<threshold:
                在 point 生成奖励物及反馈特效
            否则:
                minimum=DIFF[3.5,4,4.5,5.5,6.5]+stage*0.1
                SHOT_AT(point,vis=0x0f,id=0x16,heading,
                        speed=RAND[minimum,minimum+2])
        lifetime=sourceSpeed；D==0 时 lifetime/=2
        t==lifetime: 退出

0x153（DelayedOwnerDamage）:
    可击子部件死亡且特殊状态未阻止伤害转交时创建。
    每帧扣 max(1,child.maxHp/30)，约30帧把对应伤害逐步扣到指定 owner，
    防止子部件爆炸帧直接破坏父状态机时序；完成或owner失效后退出。

共享 HP 代理 0x151/0x152:
    固定跟随 owner 偏移，不绘制；每帧让 proxy.hp 与 owner.hp 取低值同步。
    代理被打到0等价于 owner HP 归零；owner失效时代理退出。
```
