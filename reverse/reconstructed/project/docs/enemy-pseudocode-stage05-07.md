# Stage 5-7 敌人运动与发弹逻辑伪代码

本文总结重建工程中 Stage 5、6、7 实际可生成的全部敌人实体。它描述的是敌人状态机、运动、发弹和退出条件；子弹生成后的通用运动由子弹逻辑文档统一说明。

## 记号

```text
D := 难度索引，0..4
DIFF[a,b,c,d,e] := 按 D 取表项
t := 当前实体的 helperTimer；进入新状态时通常重置为 0
T := 根 Boss 的 helperTimer
cycle(n) := t mod n
AIM(x,y) := 从 (x,y) 指向自机的 16 位角
TURN(angle,target,step) := 沿最短圆周方向、每帧最多 step 地逼近 target
MOVE(angle,speed) := (x,y) += (cos(angle),sin(angle))*speed
FAN(visual,projectileId,pos,angle,speed,count,spread) := 以 angle 为中心生成 count 发、总张角 spread 的扇形
RING(...) := FAN(..., spread=0x10000)
RAND(seed) := 原版脚本的确定性 MT 风格随机值，不是每次运行重新播种
INSIDE(m) := 位于随自机横向偏移的 600x720 游戏区，外扩 m
```

`0x4000` 为向下，`0xc000` 为向上，`0x0000/0x8000` 为向右/向左。下文简写的 `type X/sub Y` 指子弹构造参数 `visual=X, projectileId=Y`，不是敌人的 `spawnType`。除非单独说明，普通敌人 HP 归零时生成对应奖励、`0x137` Counter 实体和爆炸后停用；越过各自的外扩边界时静默停用。关卡实际调度见 `src/stage_runtime_stage05_07.cpp:68-498`，统一更新/发弹分派见 `src/stage_runtime.cpp:4755-5066`。

## 多关卡复用杂兵

### `0x07/0x08`：减速直行的大型标记敌人

Stage 5-7 都会调度该族。两者运动和发弹完全相同；类型只影响贴图以及在特定路线设置下是否生成 `0x09` Help 子机。

```text
state 0:
    显示 20 帧入场标记；将坐标夹到 INSIDE(50)
    t == 20:
        speed = 6
        按 effectiveHelpLevel() 生成若干 0x09（仅适用的 type/route 组合）
        state = 1；可被攻击

state 1:
    t <= 120 且 speed > 1: speed -= 0.1
    secondaryAngle 缓慢追踪 AIM(x,y+75)，每个射击周期首帧重新锁定
    MOVE(sourceAngle,speed)

    start = DIFF[40,30,30,30,30]
    period = DIFF[100,80,60,60,50]
    count = DIFF[1,3,5,5,7]
    若 start <= t < 240 且 local=(t-start)%period < 30 且 local%5==0:
        speedShot = DIFF[4,6,8,9.5,11] + local*0.05
        FAN(0,0,(x,y+75),secondaryAngle,speedShot,count,20000)
        若 D>2，再发一层 speedShot-2
```

`0x09` 围绕父实体作逐渐展开的椭圆轨道，不发弹；父死亡时自身爆炸并掉落 Help，父失活时直接退出。源码：`src/stage_runtime.cpp:9640-9765,11566-11595`。

### 关底 Boss `0x13c + 0x54..0x56`

根状态流程：

```text
0: 240帧入场 -> 1(HP60000,time1500)
1 -> 2过渡180帧 -> 3(55000,1500)
3 -> 4过渡 -> 5(55000,1500)
5 -> 6过渡 -> 7(70000,1800)
7 -> 8过渡；t=210生成0x54；t=380 -> 9(45000,1800)
9 -> 10过渡180帧 -> 11(80000,2100)
11: t=60生成左右0x56；t=450 -> 12
12: 60帧正弦移动到自机上方 -> 13
13: 低速移动，t=780回到11(t=90)，形成最终循环
最终击破 -> 14；240帧爆炸，480帧退出
```

根在战斗状态每50帧选择一个 `[210,510]x[150,250]` 目标，以 `0x200` 转向、通常 speed `0.3`（最终循环 `0.1`）。状态1/3/5/7的火力：

    speed=DIFF[5.7,7,7.5,8,9.5]+局部增量
    后半循环从三个枪口依次发逐渐收窄的瞄准扇

state3:
    T>=240：前两个枪口在510帧循环中发层数递增的瞄准扇
    另两个侧枪在240帧循环中预警后发 type0x11/sub0x0c 横向弹；D>2加斜向两发

state5:
    左右枪每 DIFF[22,16,12,13,11] 帧发反向旋转整环
    下方两枪交错发窄瞄准扇与宽半圆扇

state7, cycle=(T-30)%880:

## 公共不可见实体 `0x137/0x153`

### `0x137`：敌人死亡后的奖励或 Counter 弹发射器

普通敌人的 `spawnEnemyDeathRewardBurst()` 会在死亡位置生成该实体，Boss 阶段击破路径也会直接生成；只有 `counterMode==1` 时运行，否则立即退出。它不移动、不显示、不可击。

```text
interval = DIFF[5,4,3,2,2]
若 player.score!=0，则每 interval 帧、且中心y<600:
    在击破半径内选随机点 p
    heading = AIM(p) + RAND[-1000,+1000]
    threshold = RAND[180,480]
    若 distance(p,player) < threshold 且 <=480:
        在p生成奖励物与反馈特效
    否则:
        minSpeed = DIFF[3.5,4,4.5,5.5,6.5] + selectedStage*0.1
        从p发 visual=0x0f、projectileId=0x16 的 Counter 弹，
        speed=RAND[minSpeed,minSpeed+2]

lifetime = 普通敌人死亡时 enemy.radius*0.6；Stage 5-7 Boss 阶段击破时显式传140
Easy时 lifetime/=2；到期退出
```

源码：`src/stage_runtime.cpp:14954-15030,15138-15243`。

### `0x153`：延迟所有者伤害

```text
每帧:
    owner = 按 parentEntityId + parentSpawnType 查找
    若 owner 不存在/已退出: 自身退出
    chunk = max(1,maxHp/30)
    damage = min(remainingHp,chunk)
    remainingHp -= damage
    owner.hp -= damage
    remainingHp < 1 时退出
```

因此所有“击破挂载件后伤害根”的效果都是约30帧渐进生效，而非击破帧瞬间扣除；这也是多部件 Boss 阶段切换不能只看单帧根 HP 的原因。源码：`src/stage_runtime.cpp:9107-9124`。
